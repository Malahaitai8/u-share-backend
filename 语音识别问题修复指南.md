# 语音识别问题修复指南

## 问题现象

在语音识别界面，点击录音按钮后显示：**"服务暂不可用"**

## 诊断结果

通过运行 `python check_voice_service.py` 发现了以下问题：

### ❌ 问题 1: 百度 API 密钥未配置

`.env` 文件中的 `BAIDU_API_KEY` 和 `BAIDU_SECRET_KEY` 未正确配置（仍为示例值）

### ❌ 问题 2: FFmpeg 未安装

系统中没有安装 FFmpeg，导致无法将 WebM 音频转换为 WAV 格式

### ❌ 问题 3: Python 依赖包缺失

缺少 `pydub` 和 `python-dotenv` 包

### ❌ 问题 4: NLP 服务未正常启动

服务返回 404 错误，可能是未启动或配置错误

---

## 🔧 修复步骤

### 步骤 1: 获取百度 API 密钥

1. **访问百度智能云控制台**
   - 打开 https://console.bce.baidu.com/
   - 登录你的百度账号（如果没有请先注册）

2. **创建语音识别应用**
   - 点击左侧菜单 "产品服务" → "人工智能" → "语音技术"
   - 点击 "创建应用"
   - 填写应用名称（如：u-share-voice）
   - 勾选 "语音识别" 服务
   - 提交创建

3. **获取密钥**
   - 在应用列表中找到刚创建的应用
   - 复制 **API Key** 和 **Secret Key**

4. **配置到 .env 文件**
   - 打开 `services/nlp_service/.env` 文件
   - 将你获取的密钥填入：
   ```env
   BAIDU_API_KEY=你的真实API_Key
   BAIDU_SECRET_KEY=你的真实Secret_Key
   ```

### 步骤 2: 安装 FFmpeg

#### Windows 系统（你的情况）

**方法 1: 使用 Chocolatey（推荐）**
```powershell
# 如果已安装 Chocolatey
choco install ffmpeg

# 如果没有 Chocolatey，先安装 Chocolatey
# 以管理员身份运行 PowerShell，执行：
Set-ExecutionPolicy Bypass -Scope Process -Force; [System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072; iex ((New-Object System.Net.WebClient).DownloadString('https://community.chocolatey.org/install.ps1'))

# 然后安装 ffmpeg
choco install ffmpeg
```

**方法 2: 手动安装**
1. 访问 https://ffmpeg.org/download.html
2. 点击 Windows 图标
3. 下载 "ffmpeg-release-essentials.zip"
4. 解压到某个目录（如 `C:\ffmpeg`）
5. 添加到系统 PATH：
   - 右键 "此电脑" → "属性" → "高级系统设置"
   - 点击 "环境变量"
   - 在 "系统变量" 中找到 "Path"，点击 "编辑"
   - 点击 "新建"，添加 `C:\ffmpeg\bin`
   - 确定保存
6. 重启命令行窗口验证：`ffmpeg -version`

### 步骤 3: 安装 Python 依赖

在项目根目录运行：

```bash
# 切换到 NLP 服务目录
cd services/nlp_service

# 安装依赖
pip install -r requirements.txt
```

或者单独安装缺失的包：

```bash
pip install pydub python-dotenv
```

### 步骤 4: 重启 NLP 服务

```bash
# 确保在 services/nlp_service 目录
cd services/nlp_service

# 启动服务
uvicorn app.main:app --reload --port 8083
```

看到以下输出表示启动成功：
```
INFO:     Uvicorn running on http://127.0.0.1:8083 (Press CTRL+C to quit)
INFO:     Started reloader process
INFO:     Started server process
INFO:     Waiting for application startup.
INFO:     Application startup complete.
```

### 步骤 5: 验证修复

1. **运行诊断脚本**
   ```bash
   python check_voice_service.py
   ```
   应该看到所有检查都通过：
   ```
   ✅ 所有必要的环境变量都已配置
   ✅ FFmpeg 已安装
   ✅ 所有 Python 依赖都已安装
   ✅ NLP 服务正在运行
   ✅ 百度 API 认证成功
   ```

2. **测试语音识别**
   - 启动前端项目
   - 进入语音识别页面
   - 点击录音按钮
   - 说出垃圾名称（如"塑料瓶"）
   - 应该能正常识别并返回分类结果

## 为什么你同学的电脑可以正常使用？

可能的原因：

1. **她已经配置了正确的百度 API 密钥**
   - 她可能使用的是自己的百度账号密钥
   - 或者之前项目里有配置过

2. **她的电脑已经安装了 FFmpeg**
   - 可能之前做过其他视频/音频相关项目
   - 或者系统预装了相关工具

3. **她已经安装了所有 Python 依赖**
   - 可能她运行过完整的 `pip install -r requirements.txt`

4. **环境配置文件的差异**
   - Git 通常会忽略 `.env` 文件（在 `.gitignore` 中）
   - 所以每个人需要单独配置自己的 `.env` 文件

## 常见问题

### Q: 我没有百度账号怎么办？

A: 可以免费注册百度账号，语音识别 API 有免费额度（每天 50,000 次调用）

### Q: FFmpeg 安装后还是提示未找到？

A: 
1. 确保已重启命令行窗口
2. 运行 `ffmpeg -version` 验证是否在 PATH 中
3. 如果还不行，重启电脑

### Q: 服务启动后仍然报错？

A: 
1. 检查 8083 端口是否被占用
2. 查看后端日志中的错误信息
3. 确认 `.env` 文件中的密钥没有多余的空格或引号

### Q: 浏览器提示无法访问麦克风？

A: 
1. 检查浏览器是否允许麦克风权限
2. Chrome: 地址栏左侧的锁图标 → 网站设置 → 麦克风 → 允许
3. 某些浏览器只在 HTTPS 或 localhost 下允许麦克风访问
